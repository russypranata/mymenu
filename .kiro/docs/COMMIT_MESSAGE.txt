fix: correct subscription history display for trial subscriptions

## Problem
Trial subscriptions were incorrectly displaying as "Paket Bulanan" 
(Monthly Package) in the subscription history, when they should 
display as "Trial Gratis" (Free Trial).

## Root Cause
1. Migration 029 backfill logic checked `status` instead of `origin`
2. Components checked `plan_type` without considering `origin` field
3. No centralized helper functions for subscription display logic

## Solution

### Database Layer
- Updated migration 029 backfill to check `origin` field
- Created migration 030 to fix existing trial records
- Set `plan_type = 'trial'` for all `origin = 'trial'` records

### Application Layer
- Created `src/lib/subscription-helpers.ts` with centralized functions:
  - getPlanLabel(planType, origin)
  - getPlanBadgeClass(planType, origin)
  - getPlanAmount(planType)
  - getPlanDuration(planType)
  - getDaysUntilExpiry(expiresAt)
  - isSubscriptionActive(expiresAt)
- Updated components to use helper functions:
  - subscription-history.tsx
  - subscription-section.tsx
- Added active/expired status indicators

### Documentation
- Created comprehensive system documentation
- Added production fix script
- Created unit tests for helper functions
- Added before/after comparison guide
